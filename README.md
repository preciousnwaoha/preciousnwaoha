<details>
<summary> AI Engineering Notes</summary>

- Prompt = System Prompt + Context Prompt = Role & Identity, Context, Task, Input, Output, Constraints/Refusals/Rules, Behavioural/Style/Cultural Context, and Tool Descriptors & Reminders (Time, Date, Language...). 
- Refusing well is harder than answering well. Be short, Explain why (briefly), Offer an alternative ("here's a helpline").
- A mediocre answer in 2 seconds beats a perfect answer in 20. Stream responses. Parallelize tool calls. Cache aggressively. Lowest bounce.
- Evaluate risk pre-output & post-output with fallbacks (it can save costs).
- If the AI responds up to 3 times in a row without a custom tool call, then it's just another ChatGPT.
- For LLM Provider Abstraction & Cost Reductrion, Do:
    - Model pooling & Model config per provider.
    - LLM Provider Config with Fallbacks.
    - Dynamic System Prompts, & Params (E.g Max Token, Schema)
    - LLM Routing based on task and config.
    - Prompt Budget Enforcement, Chunking & Compacting.
    - Load tool catalog on-demand, not all at once
